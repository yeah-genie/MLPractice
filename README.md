# 유튜버 히아투스 레이더

> "사라지기 전에 이미 신호가 있었다" — 6개월+ 휴재 유튜버의 업로드 패턴을 분석합니다.

---

## 실행 순서

```bash
pip install -r requirements.txt

python collect.py              # ① 영상 메타데이터 수집 (구독자 30만+, 6개월+ 휴재 필터)
python clean.py                # ② 분석 지표 생성 → data/cleaned/*_videos.csv
python analyze_transcripts.py  # ③ 자막 키워드 분석 → data/cleaned/*_transcripts.csv  (선택)
```

Power BI Desktop에서 `data/cleaned/` 폴더 안의 CSV를 임포트합니다.  
`*_videos.csv` 와 `*_transcripts.csv` 는 **video_id** 를 공통 키로 연결하면 됩니다.

---

## 수집하는 데이터

| 컬럼 | 설명 | 왜 필요한가 |
|------|------|------------|
| `days_since_prev` | 이전 영상 대비 경과일 | 업로드 주기 변화 추적 |
| `duration_min` | 영상 길이(분) | 짧아질수록 "대충 올리는 신호" |
| `title_length` | 제목 글자 수 | 제목이 짧아지면 노력 감소 신호 |
| `desc_length` | 설명란 글자 수 | 설명란이 비어가면 관리 포기 신호 |
| `tag_count` | 태그 개수 | 태그를 안 달기 시작하면 무관심 신호 |
| `like_ratio` | 좋아요 / 조회수 | 팬들의 "적극적 반응" 지표 |
| `comment_ratio` | 댓글 / 조회수 | 커뮤니티 활성도 지표 |
| `risk_score` | 히아투스 위험 점수 (0~100) | 여러 신호를 종합한 경보 지수 |

## 분석 기법 설명 (초보용)

### 1. 이동평균 (Rolling Average)

**한 줄 요약:** 들쭉날쭉한 그래프를 부드럽게 다려주는 기법.

예를 들어 어떤 유튜버의 업로드 주기가 이렇다고 해봅시다:
```
7일 → 8일 → 6일 → 30일 → 9일 → 7일
```

30일은 분명 이상한데, 꺾은선 그래프로 보면 그냥 하나의 튀는 점처럼 보입니다.
4개 이동평균을 내면:
```
7 → 7.5 → 7 → 12.8 → 13.3 → 13.3
```
이렇게 보면 "중간 즈음부터 업로드가 느려지기 시작했다"는 트렌드가 보입니다.

Power BI에서는: 꺾은선 그래프에 `rolling_gap` 을 x축=`published_at` 으로 그리면 됩니다.

---

### 2. Z-score 이상탐지

**한 줄 요약:** "평소 대비 얼마나 이상한가"를 숫자로 표현한 것.

공식: `Z = (현재값 - 평균) / 표준편차`

| Z값 | 의미 |
|-----|------|
| 0 근처 | 평소와 비슷 |
| +2 이상 | 평소보다 **매우 높음** (업로드 주기라면 🔴 위험) |
| -2 이하 | 평소보다 **매우 낮음** (조회수라면 🔴 위험) |

예: 평소 7일마다 올리던 유튜버가 갑자기 60일 공백을 만들면 → `z_gap` 이 +3~4 수준으로 폭등.

Power BI에서는: `z_gap` 을 꺾은선으로, 빨간 참조선을 y=2에 그으면 "이상징후 구간"이 한눈에 보입니다.

---

### 3. 히아투스 위험 점수 (risk_score)

**한 줄 요약:** 여러 신호를 합산해서 "이 영상 올릴 때 유튜버가 지쳐있었나"를 0~100으로 표현.

계산 방식:
```
위험 점수 =
  업로드 주기 Z-score × 40%   (오래 쉴수록 위험)
+ 조회수 하락 Z-score  × 35%   (조회수 떨어질수록 위험)
+ 좋아요율 하락 Z-score × 25%  (팬 반응 식을수록 위험)
```

점수가 70 이상인 구간 = "명확한 이상징후 시기"

Power BI에서는: 영역 차트(Area Chart)로 `risk_score` 를 시간순으로 그리면 "언제부터 위험했는지"가 시각적으로 강렬하게 보입니다.

---

### 4. 정규화 타임라인 (months_before_last)

**한 줄 요약:** 채널마다 활동 기간이 달라도, "마지막 영상 기준 몇 달 전"으로 시간축을 맞춰서 여러 채널을 한 화면에 비교하는 기법.

예:
- A 채널 마지막 영상: 2023년 5월
- B 채널 마지막 영상: 2024년 2월

이 둘을 같은 그래프에 그리면 날짜가 달라서 비교가 안 됩니다.
`months_before_last = 0` 을 각자의 "마지막 영상"으로 맞추면
→ "마지막 영상 기준 -6개월부터 업로드 주기가 어떻게 변했는가"를 함께 볼 수 있습니다.

Power BI에서는: x축 = `months_before_last`, 범례 = `channel_name` 으로 꺾은선을 그리면 채널 비교 차트 완성.

---

## Power BI에서 만들 차트 목록

| 차트 | x축 | y축 | 목적 |
|------|-----|-----|------|
| 꺾은선 | `year_month` | `rolling_gap` | 업로드 주기 트렌드 |
| 꺾은선 + 참조선 | `year_month` | `z_gap` | 이상징후 시점 탐지 |
| 영역 차트 | `year_month` | `risk_score` | 위험도 타임라인 |
| 꺾은선 (다채널 비교) | `months_before_last` | `rolling_gap` | 채널별 패턴 비교 |
| 산점도 | `rolling_gap` | `rolling_views` | 주기↑ vs 조회수↓ 상관관계 |
| 막대 | `year_month` | `gap_label` (count) | 월별 이상징후 빈도 |
| 꺾은선 | `year_month` | `duration_min` | 영상 길이 변화 |

## 채널 추가 방법

`collect.py` 의 `CANDIDATE_NAMES` 리스트에 채널명을 추가하면 됩니다.
구독자 30만 미만이거나 최근 6개월 내 업로드한 채널은 자동으로 제외됩니다.

```python
CANDIDATE_NAMES = [
    ...
    "추가할채널명",   # ← 여기에 추가
]
```

## 자막 분석 — 영상 안에서 신호 찾기

`analyze_transcripts.py` 는 영상의 자막 텍스트를 읽어서 아래 5개 카테고리 키워드를 분석합니다.

| 카테고리 | 예시 키워드 | 신호 의미 |
|---------|-----------|---------|
| 번아웃 | 쉬고싶다, 지쳤어, 힘들어, 번아웃, 슬럼프 | 가장 직접적인 지침 신호 |
| 사과_지연 | 오랜만이에요, 미안해요, 기다리셨죠 | 업로드 간격 의식하는 신호 |
| 불확실_고민 | 모르겠어요, 고민 중, 앞으로 어떻게 | 방향성 잃어가는 신호 |
| 건강_피로 | 아파요, 피곤해, 스트레스, 잠을 못잤어요 | 컨디션 악화 신호 |
| 감사_작별 | 감사합니다, 함께해줘서, 사랑해요 | 마지막 인사 뉘앙스 |

### 생성 컬럼

| 컬럼 | 설명 |
|------|------|
| `번아웃_count` | 해당 영상에서 키워드 등장 횟수 |
| `번아웃_per1k` | 1000자당 등장 빈도 (길이 보정값) |
| `verbal_risk_score` | 5개 카테고리 종합 언급 위험 점수 (0~100) |
| `rolling_verbal_risk` | 4개 영상 이동평균 (트렌드 파악용) |

### Power BI 연결 방법

1. `*_videos.csv` 와 `*_transcripts.csv` 를 각각 임포트
2. 모델 뷰 → `video_id` 컬럼으로 두 테이블 관계 연결
3. 이제 하나의 꺾은선에 `risk_score` (행동) 와 `rolling_verbal_risk` (언급) 을 함께 표시 가능

> **자막이 없는 영상** (비공개, 자막 비활성화 등) 은 자동으로 건너뜁니다.  
> 자동 생성 자막도 분석하므로 대부분의 한국어 영상에서 동작합니다.
